# Plans

Plans are organized by what the document is, not by which workflow produced it.

- `design/`: product/design decisions, requirements, accepted scope, and resolved tradeoffs.
- `implementation/`: executable task plans, agent checklists, migration plans, and validation plans.

Historically, the superpowers workflow has been the sole producer of implementation plans, and
older commits may refer to `docs/superpowers/plans/` or `docs/superpowers/specs/`. Those docs have
been migrated here so the directory structure describes the document type instead of the tool that
produced it.
