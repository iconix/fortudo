# Fortudo PWA: Installable and Offline-Friendly

## Pyramid Summary

**~2w:** Fortudo PWA.

**~4w:** Make Fortudo installable, offline-capable.

**~8w:** Manifest, hand-rolled service worker, vendored assets.

**~16w:** Three tiers: installable shell, fully offline assets (Tailwind built statically), offline UX polish with reconnect sync.

**~32w:** The data layer is already offline (PouchDB local-first); this adds the delivery half: manifest and icons, a hand-rolled versioned service worker with guard-tested precache, vendored CDN dependencies, update toast, and reconnect sync polish.

## Problem

ROADMAP.md vNext: "make Fortudo a PWA: installable and offline-friendly as a first-class web app." Today there is no manifest, no service worker, no installable icons, and three render-blocking CDN dependencies. Everything else is already offline-ready: all features except CouchDB sync work with zero network (PouchDB/IndexedDB per room, localStorage room entry, no app-level fetch calls).

## System Context (research: ~/.claude/thoughts/shared/research/2026-07-07-fortudo-pwa-readiness.md)

- Local-first data: `storage.js` opens `fortudo-${roomCode}` on IndexedDB; with `COUCHDB_URL = null` the app never touches the network.
- Sync is one-shot `replicate.to`/`.from` with a 2s write debounce and focus-triggered re-sync; no retry, no online/offline listeners; failures render a red "Error" status.
- CDN deps in `index.html`: Font Awesome CSS (cdnjs), Tailwind JIT runtime (cdn.tailwindcss.com, not-for-production), PouchDB 9.0.0 (jsdelivr). PouchDB is already a devDependency and the E2E suite already serves the local copy.
- No build step: `public/` is served as-authored; CI's only mutation is rewriting `js/config.js` from a secret. No cache-busting or Cache-Control headers exist; `firebase.json` has CSP + SPA rewrite only.
- Reload-to-apply pattern: the Activities toggle relies on `window.location.reload()` (settings-renderer.js) — the established UX for "changes apply after reload."

## Decisions

1. **Tailwind: standalone CLI, committed CSS.** `tailwindcss` as devDependency; `npm run build:css` emits committed `public/css/tailwind.css`; CI freshness check (`build:css && git diff --exit-code`). Preserves serve-as-authored for local dev and E2E.
2. **Service worker: hand-rolled.** ~80 lines, versioned cache, committed script-generated precache list, guard test failing on drift (same culture as the existing AST guards).
3. **Icons: rendered 💪🏾 emoji** on the brand teal background, generated by a committed script (192/512 + maskable + apple-touch-icon). Keeps the established identity, zero design effort.
4. **Update UX: toast with Reload button.** SW `updatefound` → toast-manager toast → `skipWaiting` + reload on `controllerchange`. Matches the reload-to-apply pattern; no surprise refresh (form/timer-edit state lives in the DOM).

## Design: three tiers, one PR each

### Tier 1 — Installable

- `public/manifest.webmanifest` (name, `start_url: /`, `display: standalone`, teal theme/background, icon entries) + `<link rel="manifest">`, `theme-color`, `apple-touch-icon` in `index.html`.
- `public/icons/` generated by a committed script; binaries committed.
- `public/sw.js` v1: precache same-origin shell (fetched `cache:'no-cache'` so a new versioned cache can't be filled with HTTP-cache-stale bodies); cache-first for precached paths; **network-first for `js/config.js`** (CI-rotated CouchDB URL must not go stale), which is also **seeded into the cache at install** — app.js statically imports it, so offline boot dies without it and the runtime handler alone never caches it during the first (uncontrolled) visit; passthrough for everything else. Registered from `index.html` with `updateViaCache: 'none'`.
- Vendor PouchDB to `public/vendor/pouchdb/pouchdb.min.js` from the devDependency (npm script) — under `public/vendor/`, not `public/js/`, because the lint glob would trip on a `.min.js` there; drop the jsdelivr tag; CSP `script-src` loses jsdelivr. Follow-up unlock: `install_local_pouchdb_route` and its guard become deletable from the E2E suite.
- `firebase.json`: `Cache-Control: no-cache` for `sw.js` and `sw-precache.js` (a cached SW cannot update; discovery depends on these revalidating).
- Precache list + content-hash cache version generated by npm script (dotfiles skipped), committed, drift-guarded by `tests/test_sw_precache_guard.py`; the generator also stamps the version into `sw.js` itself so the main SW script changes every deploy.
- Result: installable everywhere; offline NOT yet achieved (Tailwind/Font Awesome still CDN — accepted).

### Tier 2 — Offline-capable assets

- Tailwind static build per Decision 1; delete the CDN runtime tag (also removes the largest render-blocking script).
- Vendor Font Awesome CSS + woff2 subset; delete cdnjs tags; tighten CSP to `'self'` + CouchDB `connect-src`.
- Precache list covers vendored/generated assets; cache version = content hash of all precached files, so any deploy rolls the cache.
- Update toast per Decision 4.
- Result: cold-start with zero network works end to end.

### Tier 3 — Offline UX polish

- `online` event listener → `triggerSync({ respectCooldown: false })` (registered alongside `focus` in `app-lifecycle.js`'s `start({ signal })`; skips the cooldown deliberately — reconnect is an explicit signal).
- Distinct "Offline" sync status (gray) via `navigator.onLine`, replacing the misleading red "Error" when the network is simply absent.
- `navigator.storage.persist()` at boot — protects IndexedDB-resident app data from Safari eviction on installed PWAs.
- `tests/e2e/test_pwa.py`: SW registration, manifest served, offline reload flow via `context.set_offline(True)` after first load. Fresh browser profiles per test mean the SW is exercised but never stale across tests.

## Testing Notes

- Playwright route interception does not see requests made by a service worker; harmless because Tier 1 removes the only intercepted route (PouchDB CDN).
- Guard tests: SW precache drift, Tailwind CSS freshness (CI), plus existing collection-hygiene/boundary guards unchanged.

## Out of Scope

Lighthouse CI, push notifications, Background Sync API, custom icon artwork, live/continuous PouchDB sync.
